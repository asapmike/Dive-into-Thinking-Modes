The answer developed across the deck is that the extra tokens create additional decoding steps and a reusable workspace.

Direct answers have no separate workspace.

Internal activations still exist, but there is no dedicated sequence of pre-answer tokens.

Most task computation must fit inside the fixed-depth passes that produce the answer.

A direct response can contain several answer tokens. What it lacks is a separate scratchpad before committing to the answer.

1 minute. Direct answer does not mean zero computation. The Transformer still builds hidden activations. The limitation is that the model cannot choose a long pre-answer workspace. Each answer prediction passes through the same fixed stack of layers.

Reasoning tokens become a workspace.

Each generated reasoning token adds another decoding step through the Transformer layers.

Question → Reason 1 → Reason 2 → Answer

Few-shot CoT: show worked reasoning examples in the prompt.

Zero-shot CoT: add a trigger such as “Let’s think step by step.”

No parameter update: this is prompt engineering at inference time.

1 minute 30 seconds. Distinguish the papers carefully. Wei et al. use few-shot reasoning demonstrations. Kojima et al. introduce the zero-shot trigger. With KV caching, the whole context is not recomputed from scratch, but each new token still travels through all Transformer layers and becomes available to later tokens.

Instruction or demonstrations at inference time

Trained reasoning

SFT, distillation, or reinforcement learning

Prompting elicits a behavior without updating the model.

Prompting examples PaLM and GPT-3 with reasoning demonstrations
Training examples Quiet-STaR, DeepSeek-R1, OpenAI o-series
1 minute 30 seconds. This slide answers the central distinction. A prompt changes the context. Reasoning training changes the parameters and the model's default behavior. Quiet-STaR is a clean research example of training a model to generate thoughts that improve future prediction. Mention DeepSeek-R1 and the o-series only as modern examples, without claiming their full proprietary architecture is known.

Hidden is about visibility, not representation.

The same kind of reasoning token can be exposed, summarized, or kept private.

Explicit reasoning
x → z language tokens → y

User sees the reasoning and the answer.

Hidden reasoning
x → z private tokens → y

User sees an answer or a model-generated summary.

Displayed CoT is not guaranteed to be a faithful recording of every factor that caused the answer.
1 minute 15 seconds. Hidden reasoning is not automatically a new representation. It may still use discrete language-like tokens. The difference is what the interface reveals. OpenAI says the raw o1 chain is not shown and a model-generated summary may be shown instead. Also warn that a displayed rationale is not a perfect brain recording.

Reasoning does not have to be language.

Token CoT

ht → words → embedding → ht+1

Latent reasoning

ht → ht+1 → ht+2

Potential: richer continuous states and fewer verbose tokens.

Cost: harder to interpret, supervise, and verify.

Status: an active research direction, not a standard deployed mechanism.

Hidden describes visibility. Latent describes representation.
1 minute 30 seconds. Both visible and hidden CoT can still be discrete tokens. Coconut changes the representation itself. It feeds the last hidden state back as the next input embedding between special boundary tokens. Emphasize the potential and the current limitations equally.
